## Decode-path latency (gap 2, Phase 0)

The visualise-modal-open hot path today is:

```
MarkedTrace.parse(stdout)                    // marker extraction
  → decodeTrace(json)
    → js.JSON.parse(json)                    // native V8/JavaScriptCore parser
    → hand-coded structural walk over steps + heap (HeapTrace assembly)
```

Phase 2 will replace `js.JSON.parse` + the hand walk with `circe.decode[HeapTrace](json)`
on Scala.js. Historically that's 10–40× slower than `js.JSON.parse` for similar inputs.
If Phase 2 measures past 2× this baseline, the plan loops back to keep `js.JSON.parse`
on the JS side with codegen-generated runtime validators (`zod` / `io-ts` from the
viz-schema codegen).

### Synthetic 500-step trace

Generated by `client/test/bench/decode-latency.bench.ts` — a 500-step trace whose
each step holds 4 heap objects (one `Arr` with 3 `Ref` cells + 3 `Instance` nodes
with two scalar fields). Total: 8 000 heap objects walked. The plan called for
~30 KB; the actual synthesised JSON is ~680 KB which is closer to a real `viz`
modal payload (real `graph-bfs` is 2.3 KB/step × ~500 steps ≈ 1.1 MB).

Run with:

```bash
cd client && npm run test:bench
```

### Notes on environment

- Run in Node 20 via vitest 4 bench mode (Node's V8 `JSON.parse` is the same engine
  code shipped in Chrome — the browser-side number is structurally equivalent, not
  identical: Safari/Firefox engines differ by < 20% in `JSON.parse` micro-benchmarks).
- Numbers are HARNESS (Node), not browser. The Phase 2 follow-up will add an
  in-browser Playwright bench so the comparison is apples-to-apples.
- The structural walk in the bench mirrors the SHAPE of `decodeTrace` → `decodeStep`
  → `decodeFrame` → `decodeValue` / `decodeObject` (one type-switch per value).
  It does NOT actually emit Scala `HeapTrace` case classes (Scala.js side does that
  in Scala-emitted JS); the walk's TS cost is a close proxy for the equivalent JS
  emitted by Scala.js, which is what production runs.

### Phase 2 acceptance criterion

> p99 of `circe.decode[HeapTrace]` on the same 500-step synthetic ≤ **13 ms**
> (= 2× Phase 0 p99 of 6.57 ms).

If Phase 2 exceeds this, the plan loops back to keep `js.JSON.parse` + generated
runtime validators (option discussed in `~/.claude/plans/before-we-proceed-further-swirling-storm.md`
gap 2 mitigation).
